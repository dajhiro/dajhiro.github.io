### 특징
- JSON 파일 작성
- 어떤 도구를 쓸지 자동으로 판단
- [[MCP]] 지원
- 진입장벽 높음
	- 1~2주

### 과정
- Node
- 템플릿 라이브러리: 미리 만들어진 워크플로우
	- 예시로서의 역할
	- 불러다가 수정하면 됨
- API 사용해야함: OpenAI API 등
	- 폴백 사용
- 예시:
	- RSS 뉴스 ⇒ AI 요약 ⇒ Slack 발송
	- 고객 문의 ⇒ 자동 분류 ⇒ 담당자 라우팅
	- 새 블로그 발행 ⇒ SNS 자동 포스팅
- 버그 처리
	- 에러 로깅을 통해
- AI Agent Node
	- AI가 직접 여러 도구 중에서 고름

### 대표적 케이스
- 콘텐츠 인텔리전스 파이프라인: 수집 ⇒ 요약 ⇒ 초안 ⇒ ==승인요청== ⇒ 발행
- 고객 지원 자동화: 문의 ⇒ AI 분류, 답변 생성 ⇒ 복잡한 건 사람에게 에스컬레이션
- 경쟁사 모니터링: 경쟁사 홈페이지 스크래핑 ⇒ 변경사항 감지 ⇒ AI 분석 ⇒ 팀 알림
- 채용 스크리닝: 지원서 수신 ⇒ AI 1차 검토 ⇒ 기준 충족 시 면접 일정 제안 이메일 발송
- 재무 데이터 처리: 영수증/인보이스 이메일 수신 ⇒ AI OCR 처리 => 회계 시스템 입력

### 요구사항
- 구체적인 System Prompt
- 도구(Tool)는 5개까지
- 오류 처리 노드
- 메모리 노드
	- 컨텍스트(상태) 유지
- 실 데이터로 테스트

### 과정-분석
- 이벤트: 시간, 이메일, 게시글 작성 등
