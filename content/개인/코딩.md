### C에서 에러처리 방법과 철학
함수
```c
FILE *fp = fopen("data.txt", "r");
if (fp == NULL)
{
  perror("fopen failed");
  return (-1);
}
```

> 에러는 값이다

Explicitness 명시성

코드를 읽는 사람이 그 자리에서 실패 가능성과 처리 방식을 다 볼 수 있음
예외처럼 호출 스택 어딘가에서 갑자기 **점프해서 처리되는** 숨은 흐름이 없다.

암묵적인 것을 믿지 말라. no hidden magic.

대표적 패턴
1. 반환값으로 상태, 결과는 포인터로
```c
int parse_int(const char *str, int *out) {
	if (!str)
		return -1;
	*out = atoi(str);
	return 0;
}
```

2. errno - 전역(또는 스레드 지역) 에러
```c
if (write(fd, buf, len) == -1)
{
	if (errno == EINTR)
	{
		// 재시도...
	}
}
```

3. goto를 이용한 cleanup 패턴
```c
int do_work(void)
{
	int rc = -1;
	char *buf = malloc(100);
	if (!buf)
		goto out;
	
	FILE *fp = fopen("f.txt", "r");
	if (!fp)
		goto free_buf;
	rc = 0;
	
fclose_fp:
	fclose(fp);
free_buf:
	free(buf);
out:
	return (rc);
}
```

### 버그와 실패의 구분
버그: 발생하면 안됨 ⇒ `assert()`로 죽임
- Null pointer 역참조, 배열 범위 초과, 잘못된 함수 호출
- 고쳐야 하는 건 바로 터뜨려라
실패: 정상적으로 발생할 수 있음
- 파일이 없음, 메모리 부족, 네트워크 끊김
- 반환값/errno로 우아하게 처리
