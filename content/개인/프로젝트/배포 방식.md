### Docker로 실행해서 감싸야 하는 이유가 뭐야?
개발자 편의 목적
- "내 컴퓨터에선 되는데 너네 컴퓨터에선 안 됨" 문제를 없애려고
- Node.js 버전, OS 차이, 의존성 충돌 등을 격리
- 여러 서비스 (DB, Redis, 앱 서버)를 한번에 묶어서 실행

Obsidian의 경우는
Obsidian은 **Electron 기반 Desktop App**

구조
- [[Electron]] = Chromium(브라우저) + Node.js를 하나의 실행 파일로 묶은 것
- 즉 웹앱인데 브라우저 없이 독립 실행되는 것처럼 보임
- 설치 파일 (.exe / .dmg / .AppImage) 하나로 끝

Obsidian이 Docker 필요 없는 이유
- 서버가 없음 → **완전 로컬에서 실행**
- DB도 없음 → 그냥 `.md` 파일을 직접 읽고 씀
- ==네트워크 의존성== 거의 없음 (Sync, Publish 플러그인 제외)
