[[TCP]]/IP 모델
네트워킹에서 가장 기본이 되는 프로토콜 스위트

프로토콜: 규칙과 절차의 집합

4계층으로 구성
1. [[Network Interface Layer]]
2. Internet Layer
3. Transport Layer
4. [[Application Layer]]


주요 프로토콜
- TCP: 신뢰성, 연결 지향적, 데이터를 세그먼트로 나누고, 수신 측에서는 다시 조립
- UDP: 연결 없이, 데이터 전송
- IP: 주소 정보, 라우팅 정보 제공

전송 과정
- TCP
	- 송신:
		- 데이터를 세그먼트로 쪼갬
		- 각 세그먼트에 순서 번호를 부여
	- 수신:  재조립, 누락 존재시 재전송 요청
	- 흐름, 혼잡 제어:
		- 네트워크의 상태, 수신자의 처리 능력에 맞춰
		- 데이터 전송 속도 조절
		- 네트워크 혼잡 및 데이터 손실 방지
- IP
	- 데이터 패킷의 라우팅:
		- 데이터 패킷을 송신자로부터 수신자까지 전달하는 데 필요한
		- 주소 정보와 라우팅 정보를 제공
	- 다양한 라우터를 거쳐, 최종 목적지 전달

표준화
- 보편 연결성
	- TCP/IP는 다양한 네트워크 장비 사이의 통신을 가능하게 하는 표준화된 규약 제공
	- 이를 통해 웹 브라우징, 이메일, 파일 전송 등 다양한 서비스 가능
- 확장성과 유연성
	- 새로운 기술과 서비스를 쉽게 통합
	- 새로운 요구 사항 수용 가능한 유연성

사례: TCP/IP 계층 위의 응용 계층(Application Layer)
- DNS: URL 주소를 IP로 변환
- Hypertext Transfer Protocol: Request/Response
- 이메일
	- SMTP, Simple Mail Transfer Protocol: 메일 송신, 텍스트
		- MIME, Multipurpose Internet Mail Extension: 이미지 및 첨부파일
	- POP3, Post Office Protocol Version 3: 메일 수신
		- IMAP, Internet Message Access Protocol
- 파일 전송
	- FTP, File Transfer Protocol: 파일 업로드/다운로드

DNS
Application Protocol 이 TCP 연결을 맺기 전에,
IP 주소를 찾아주는 UDP 기반의 인프라 서비스
