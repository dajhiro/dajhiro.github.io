서블릿 컨테이너의 역할
- 서블릿을 통해 웹 서버와 통신하는 코드를 추상화하여 쉽게 만들 수 있게 한다.
- 소켓 생성, listen, accept 를 API로 추상화
- 비즈니스 로직에만 집중 가능

생명주기 관리
- `init()`, `service()`, `destroy()`

멀티스레드 지원
- 요청 ⇒ 스레드 풀에서 스레드 하나 할당 ⇒
	- 스레드: HTTP 서비스 메소드 절차(요청에 맞는 작업, 후 응답) 실행
	- 종료 후 스레드를 풀에 반환

선언적인 보안 관리
- 보안 설정을 코드에 직접 작성하는 대신,
- 배포 서술자(`web.xml`)에 선언적으로 정의하면 인증/인가를 처리.
- 보안 로직을 비즈니스 코드와 분리할 수 있음.

서블릿 컨테이너와 웹 서버의 차이

Tomcat과 Jetty의 차이: 핵심은 설계 철학의 차이다
- Jetty: 애플리케이션 안에 임베드
- Tomcat: 독립 실행형 서버로 사용

Spring Boot가 기본 내장 컨테이너로 Tomcat을 사용하지만, Jetty로 교체하는 것도 흔함.

역할 자체는 Jakarta EE가 동일하게 정의.
