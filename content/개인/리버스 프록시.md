[[Proxy]]
Reverse Proxy

클라이언트는 리버스 프록시를 최종 목적지로 인식
리버스 프록시는 백엔드 서버에 요청/응답 통신

목적: 로드 밸런싱, API 게이트웨이, CDN, A/B 테스트, 보안

---
L4 로드 밸런서
- 기반 레이어: Transfer Layer (TCP, UDP)
- 판단 기준: IP, 포트
- 특징:
	- 패킷 내용을 해석하지 않음 ⇒ 모든 프로토콜 지원
	- 단순 포워딩, 빠르고 가벼움
	- 한 번 연결된 백엔드는 연결이 끊길 때까지 고정
	- ==캐싱/콘텐츠 기반 라우팅 불가==
- 장점
	- 프로토콜 독립적
	- 성능 우수, 지연 적음
	- 암호화 해제 불요 (End to End TLS 유지 가능)
- 단점
	- 요청 내용 기반 분기 불가
	- 세션/요청 단위의 세밀한 로드 밸런싱 불가
	- 캐싱 불가

L7 로드 밸런서
- 기반 레이어: Application Layer (HTTP, gRPC 등)
- 판단 기준: URL, HTTP 메서드, 헤더, 쿠키 등
- 특징:
	- 요청 내용을 파싱하여 라우팅 가능
	- 특정 경로/메서드별 다른 백엔드로 분기
	- 캐싱, 압축, 인증, API 게이트웨이 기능 수행
	- TLS 종료 (TLS Termination) 가능
- 장점
	- 스마트 라우팅 가능 (마이크로서비스에 최적)
	- 캐싱 및 콘텐츠 최적화 가능
	- API 게이트웨이 역할 수행 가능
- 단점
	- 패킷 복호화 및 파싱으로 부하 증가
	- 지원하지 않는 프로토콜은 처리 불가
	- TLS 키 관리 필요
