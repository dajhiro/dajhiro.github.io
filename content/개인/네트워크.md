### 기본
TCP
UDP

혼잡 제어 알고리즘 Congestion Control

TCP : 송신 측의 데이터 전송 속도를 조절하는 매커니즘

BBR, Bottleneck Bandwidth and RTT
기존의 패킷 손실(Loss-based) 기반이 아닌,
실제 네트워크 대역폭과 지연 시간을 측정하여
속도를 개선한다.

전통적 방식
Tahoe, Reno, Cubic 등 패킷이 유실되면 전송 속도를 절반으로 줄이는 방식

QUIC과 HTTP/3
UDP 기반이지만 TCP처럼 신뢰성을 보장하도록 재설계된 프로토콜

- TCP + TLS 한계 극복: 연결 설정(Handshake) 시간을 단축(1RTT 또는 0RTT)합니다.
- 멀티플렉싱 Multiplexing : 하나의 연결에서 여러 스트림을 보낼 때, 하나의 패킷이 손실되어도 다른 스트림이 막히지 않는 head-of-line blocking 문제를 해결

### 대규모 아키텍처 및 로드 밸런싱
L4 vs L7 로드밸런서
- L4(Transport 계층): IP와 포트 번호만 보고 트래픽을 분산합니다.
- L7(Application 계층): HTTP 헤더, 쿠키, URL 패턴 등 실제 콘텐츠를 보고 분산합니다.
	- 지능적인 라우팅(예: `/api`는 API 서버로, `/static`은 이미지 서버로)이 가능하지만, 패킷을 열어봐야하므로 자원 소모가 큽니다.

로드 밸런싱 알고리즘
- Round Robin, Least Connections 외, 클라이언트 IP의 해시값을 이용해 항상 같은 서버로 연결을 보장하는 Consistent Hashing 등이 중요.

### 네트워크 가상화 및 현대적 인프라 (SDN & 가상화)
하드웨어 장비(라우터, 스위치)를 직접 만지는 것보다 소프트웨어로 네트워크를 제어하는 기술이 주류가 됨

SDN(Software Defined Networking)
네트워크의 제어부(Control Plane)와 
데이터 전송부(Data Plane)를 분리하여,
중앙 소프트웨어(컨트롤러)에서 전체 네트워크를 프로그래밍하듯 관리하는 기술

Overlay Network(VXLAN)
물리적인 L2/L3 네트워크 위에,
소프트웨어적으로 가상 L2 네트워크를 구현하는 기술.
클라우드나 쿠버네티스(Kubernetes) 환경에서
수많은 컨테이너 간의 독립된 통신을 보장.

### 고성능 네트워크 기술
소프트웨어 애플리케이션의 병목이 아닌,
커널(Kernel)과 OS 레벨에서의 네트워크 병목을 해결하는 기술

커널 우회, Kernel Bypass - DPDK/RDMA
일반적인 네트우크 통신은
- NIC(랜카드) => 커널 공간 => 유저 공간(앱)의 복사 과정을 거침
	- 오버헤드가 크다
	- DPDK/RDMA 기술을 쓰면 커널을 거치지 않고 앱이 네트워크 카드에 직접 접근

eBPF, Extended Berkeley Packet Filter
리눅스 커널 코드를 수정하지 않고도 커널 내부에서 안전하게 프로그램을 실행할 수 있는 기술,
마이크로서비스 환경에서 초고속 네트워크 보안 및 모니터링(예: Cilium)에 적극 활용

제로 트러스트 Zero Trust
Never Trust, Always Verify
매번 신원을 검증, 최소한의 권한만 부여

Signal 기반 제어
ID, 기기 보안상태, 접속 국가, 시간대 등을 분석하여 동적으로 통제
